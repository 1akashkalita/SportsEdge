## Root cause (diagnosed, NOT a model regression)

These are **stale golden-value tests**, not a model bug:

- The projection **mean** (`base = avg_l5*0.45 + avg_l10*0.35 + avg_l20*0.20`, then
  `projection = base*pf*mf`) is byte-identical since the file was first tracked
  (`867edc5`); commit `a0aa663` (when the test was committed, Jun 21) had no calibration
  import, so its mean math equals HEAD's. The hit-rate data
  (`data/research/hit_rates/nba/*.json`) is gitignored and frozen at **Jun 13** тАФ before
  the test was committed. Therefore the model produced `31.796`/`21.418` at commit time
  too: the golden anchors `32.651`/`23.905` **never matched the committed data**. Born
  failing.
- The smaller **sigma/over_probability** drift traces to `load_calibration_factor("nba")`
  reading the **mutable, gitignored** `data/research/calibration.json` (currently
  `NBA: 1.0`, neutral). Hardcoding the post-cal value would let the test re-break when NBA
  calibration computes a non-neutral factor.

All *behavioral* assertions (tier `A`/`SKIP`, EV sign, hit-rate `9/10`,
`recomputed_today_line`) already pass тАФ only the 3 numeric anchors per test are stale.

Constraint honored: **no production / model change** тАФ real-money pick outputs untouched.
