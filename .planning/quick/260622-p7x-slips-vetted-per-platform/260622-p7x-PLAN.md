# Quick Task 260622-p7x: Trustworthy, platform-specific slip generation

## Background (investigated, do not re-derive)

`build_slips.py` builds slips from ALL projections via its own weaker `is_eligible()` filter
(NOT the gate gauntlet) and hardcodes `platform="PrizePicks"` in `make_slip()`. Result observed
2026-06-22: 6/8 slips mixed Underdog+PrizePicks legs into one "PrizePicks" slip (unplayable),
one slip had a duplicate leg, and legs were not the recommended/vetted picks.

The **gauntlet-vetted universe** = APPROVED picks (workbook `Picks` sheet, Status=APPROVED)
PLUS Gate-8 cap-held picks (`Skipped Picks` sheet where `Gate Failed` contains
`"GATE 8 — DYNAMIC EXPOSURE CAP"` or `"GATE 8 — CONCENTRATION CAP"`). These passed every
quality gate and were held back only by the bankroll exposure cap — genuinely good, not random.
The MISSING-EV / MISSING-PROBABILITY "Gate 8" rows are quality hard-stops — NOT vetted.

Workbook path: `data/<sport>/<sport>_<date>.xlsx` (sports: nba, mlb). Relevant columns —
`Picks`: Sport, Platform, Player/Team, Line, Selection, Pick Type, Status.
`Skipped Picks`: Sport, Platform, Player/Team, Line, Pick, Pick Type, Gate Failed.
Projection JSON (`data/research/projections/<sport>/<sport>_projections_<date>.json`) carries
clean fields incl. real `platform`, `player_name`, `stat_type`, `pp_line`, `over_probability`,
`edge`, `expected_value`, `confidence_tier`, `flags`, `sample_size`, `line_timing`.

## Task 2 — Implement vetted filter + per-platform single-platform slips
**files:** scripts/build_slips.py (and minimal helper; may add a small canonical-name fn locally)
**action:**
  - Add `load_vetted_keys(date) -> set[tuple] | None`: load each sport workbook for `date`;
    collect vetted rows = `Picks` Status==APPROVED + `Skipped Picks` Gate Failed containing
    "GATE 8 — DYNAMIC EXPOSURE CAP" or "GATE 8 — CONCENTRATION CAP". For each, store a key of
    (sport-upper, canonical(player), float(line), platform) plus the Selection/Pick text for
    stat-token disambiguation. Return None if NO workbook exists for the date (fallback to old
    behavior for historical backfill).
  - Add `filter_to_vetted(projections, vetted) -> list`: keep a projection iff its
    (sport,canonical(player_name),float(pp_line),platform) matches a vetted key AND the
    projection's stat_type tokens are contained in that vetted row's Selection text
    (normalize: lower, replace +/_ with space, drop stopwords like over/under/inn/1st/pitcher).
    Unmatched/ambiguous → EXCLUDE (fail-safe). Canonical name = casefold + strip accents/punct.
  - In `make_slip`: derive `platform = legs[0]["platform"]` (assert all legs share it); pass the
    REAL platform to `payout_multiplier`; set `slip["platform"]` to it. If multiplier is None,
    keep the field None and let render show "n/a".
  - Refactor `build_slips()` to PARTITION the eligible pool by `platform` and run the EXISTING
    per-category combo logic (safest/highest_ev/diversified/correlated/kat) WITHIN each platform,
    appending at most one slip per (category, platform). This preserves the
    `payload['slips'][category]` list structure (existing tests index `[0]`), guarantees
    single-platform slips, and makes dedup automatic (prop_id unique per player/stat/line within
    a platform). Skip a (category,platform) when no valid >=2-leg combo exists.
  - `main()`: `projections = load_all_projections(date)`; `vetted = load_vetted_keys(date)`;
    `if vetted is not None: projections = filter_to_vetted(projections, vetted)`; then build.
    Add `platform` breakdown + `vetted_source` ("workbook" | "fallback_is_eligible") to the
    returned payload + the printed JSON_RESULT.
  - `render_markdown`: include `Platform: <name>` on each slip's header line; render None payout
    as "n/a"; optionally group output under per-platform headings. Keep markdown valid for
    `send_slips_telegram.py` (no signature change there).
  - If `analyze_prop_correlation`/`audit_slips` need a light touch to stay consistent
    (e.g. audit asserting single-platform), make the MINIMAL change; do not broaden scope.
