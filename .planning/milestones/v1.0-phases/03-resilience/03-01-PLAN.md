## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runner → spawned subprocess (fetch/hit-rate/projection) | The runner controls the argv; no untrusted input crosses, but a retried subprocess can amplify outbound API calls |
| runner → external APIs (ESPN, DFS, Telegram) via the subprocesses | Retry can repeat outbound requests; bounded by D-02/D-03 |
| OS signal → runner main thread (SIGALRM) | The signal handler runs arbitrary cleanup (subprocess kill) on the main thread |
| completed-task state → failure-alert decision | The `_task_result` flag governs whether a genuine failure is alerted (must not mask real failures) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | RES-01 retry loop amplifying outbound API requests | mitigate | Max 1 re-run (D-03); exit-0/empty-board NOT retried (D-02); 5 s backoff between attempts; retry is opportunistic within the < 120 s task budget, never stacking stage ceilings |
| T-03-02 | Denial of Service | Orphaned subprocess children after SIGALRM fires mid-`subprocess.run()` | mitigate | `_sigalrm_handler` calls `_current_subprocess.kill()` + `.wait(timeout=5)` before raising; helper uses Popen + clears `_current_subprocess` in finally so the handler always sees the live child |
| T-03-03 | Repudiation | `_task_result` flag suppressing a genuine `❌ SPORTS TASK FAILED` alert | mitigate | Flag set ONLY after `run_task()` returns successfully (outside the locks `with` block — Pitfall 5); reclassification gated on `_task_result is not None AND isinstance(e, BrokenPipeError)`; any other exception or pre-completion pipe error still alerts |
| T-03-04 | Tampering | `_sigalrm_handler` executing kill on a tampered `_current_subprocess` | accept | `_current_subprocess` is a module-private global set only by the runner's own helper; no external write path; macOS single-operator unattended cron — no untrusted local actor |
| T-03-05 | Denial of Service | Stale `.lock` / `.tmp` files after a SIGALRM timeout interrupt | mitigate | SIGALRM raises through `with` blocks whose `__exit__` releases locks (Pitfall 2, verified); `safe_save_workbook` `finally` unlinks orphaned `.tmp` and `os.replace` is atomic (D-07 interrupt-anywhere safety) |
