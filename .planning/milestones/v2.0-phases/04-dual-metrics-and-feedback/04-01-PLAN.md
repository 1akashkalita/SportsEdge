## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | load_calibration_factor reading calibration.json | mitigate | try/except wraps json parse → returns neutral 1.0 on ANY failure (missing/corrupt/malformed) — never raises |
| T-04-02 | Input Validation (V5) | factor value read from calibration.json | mitigate | clamp the read factor into [0.85, 1.20] before use even if the file holds an out-of-range value |
| T-04-03 | Tampering | write_calibration_json producing a partial/malformed file | mitigate | atomic write: serialize to `.json.tmp`, then `os.replace(tmp, path)` — no half-written file ever visible |
| T-04-04 | Denial of Service | unbounded `audit` array growth in calibration.json | mitigate | trim `audit` to the last 52 entries (~1 year) on every write |
| T-04-05 | Tampering / Elevation | calibration loop reaching grading or gate code | mitigate | calibration.py imports neither evaluate_no_bet_gates nor grading code; TestIntegrityNoGateImport (AST) asserts this structurally (METRICS-03 / D-13) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | no package installs in this phase — stdlib + openpyxl only; nothing to verify |
