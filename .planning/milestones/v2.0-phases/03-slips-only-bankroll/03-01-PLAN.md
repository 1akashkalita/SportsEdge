<!-- The complete deterministic staking rule, copied verbatim from 03-CONTEXT.md specifics.
     The executor implements EXACTLY this branch order; the EV gate is checked FIRST. -->

Staking rule (D-02..D-06), evaluated top-to-bottom:
  if combined_ev_score <= 0:            stake = 0.0                          # D-05 EV gate (sign test, not a tuned cutoff)
  elif combined_probability < 0.58:     stake = 0.0                          # D-04 zero-floor
  elif combined_probability >= 0.75:    stake = 0.025  * start_of_day_bankroll  # D-03 high tier
  elif combined_probability >= 0.65:    stake = 0.015  * start_of_day_bankroll  # D-03 mid tier
  else:  # 0.58 <= prob < 0.65          stake = 0.0075 * start_of_day_bankroll  # D-03 low tier

Round each non-zero stake to 4 decimal places. combined_ev_score is on an un-normalized model
scale (~1.47 in samples) so the gate is strictly a positivity test (<= 0), NOT a numeric threshold.

Module shape mirrors scripts/slip_payouts.py (stateless math, no runner import, no import-time side effects):
  - `#!/usr/bin/env python3`, module docstring, `from __future__ import annotations`
  - `from typing import Any`
  - `ROOT = Path(__file__).resolve().parents[1]` only if a path is actually needed (it is not here — omit if unused)

Public API:
  - confidence_stake(combined_probability: float, combined_ev_score: float, start_of_day_bankroll: float) -> float
  - apply_confidence_stakes(slips: list[dict[str, Any]], start_of_day_bankroll: float) -> list[dict[str, Any]]
      returns copies of each slip dict with "stake_units" populated from confidence_stake(); coerces
      missing/None combined_probability and combined_ev_score to 0.0 via `float(x or 0)`.
</interfaces>
