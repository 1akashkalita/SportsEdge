<objective>
Execute the one-time clean rebase of the bankroll history onto the slips-only basis from inception (BANKROLL-03 / D-11 / D-12 / D-14). This adds `rebuild_slip_bankroll()` — a chronological, idempotent function that, for each date from 2026-06-08 forward, takes the start-of-day bankroll snapshot, recomputes every graded slip's stake in place via `confidence_stake()` (re-deriving Gross Return / Net PnL through `calculate_slip_payout`, upserting by Slip ID through the existing `write_slip_history_rows`), excludes reconciliation rows (D-13), applies the day's aggregate Net PnL once at day close (no intra-day compounding, D-14), and rebuilds Daily Log + Bankroll Chart Data + `bankroll.json` with `starting_bankroll=100`. It is wired as a runner task `rebuild_bankroll`.

Purpose: This converts the historical ledger — currently prop-based (`current_bankroll = 107.892` from 295 prop units) — into a slips-only series starting 2026-06-08. The math is proven by unit tests; the actual write to the live `data/pnl/master_pnl.xlsx` + `bankroll.json` is a real-money operation, so it runs behind a human-verify checkpoint after the automated dry-run is confirmed.

Output: Edited `scripts/sports_system_runner.py` (new `rebuild_slip_bankroll(dry_run=...)` + `rebuild_bankroll` task wiring) and added tests in `scripts/test_slip_bankroll.py`. The plan ends with a human-verified production rebuild against the live ledger.

Rebuild algorithm (D-11/D-12/D-14, from 03-RESEARCH Pattern 4 + Q4):
  1. master_pnl_workbook(); locate Slip History.
  2. Wipe Daily Log + Bankroll Chart Data ONLY for the target date range (Jun 8 onward) via remove_rows_for_date
     in a loop — preserve any future dates; do the wipe + re-append then a SINGLE save_workbook_atomic (Pitfall 3).
  3. current_bankroll = 100 (starting). For each unique slip date in ASCENDING order from 2026-06-08:
       a. start_of_day = current_bankroll (D-14 snapshot — no intra-day compounding)
       b. for each slip on that date:
            - if Needs Payout Reconciliation truthy: skip (D-13)
            - new_stake = confidence_stake(combined_probability, combined_ev_score, start_of_day)
            - payout = calculate_slip_payout(..., stake_units=new_stake, ...) (re-derive multiplier)
            - upsert via write_slip_history_rows with updated stake/gross/net (idempotent by Slip ID, D-12)
       c. day_net = sum of bettable Net PnL for the date
       d. current_bankroll = round(current_bankroll + day_net, 3)  (applied once at day close, D-14)
       e. append Daily Log row + Bankroll Chart Data row [date, current_bankroll, roi, now_iso()]
  4. write bankroll.json (starting=100, current=final, last_graded_date=last slip date); refresh_prop_accuracy(wb).
  5. dry_run=True returns the final bankroll dict WITHOUT writing; dry_run=False -> save_workbook_atomic THEN bankroll.json.
