# Phase 04.1 Plan 01: Forward Confidence Staking (BANKROLL-02) Summary

**One-liner:** Wired `apply_confidence_stakes` from `stake_sizing.py` into `build_slips.main()` so daily slip builds emit real confidence-tiered stakes (2.5%/1.5%/0.75% of bankroll) instead of the flat `stake_units=1.0` placeholder.

## What Was Built

### Task 1 — TestForwardStaking RED (commit da4a09c)

Added `TestForwardStaking(unittest.TestCase)` class to `scripts/test_build_slips.py` with three test methods that pin the BANKROLL-02 stake contract:

- `test_high_prob_slip_gets_real_stake`: combined_probability=0.80, ev>0, bankroll=100.0 → asserts `stake_units == 2.5` (2.5% tier)
- `test_low_prob_or_negative_ev_gets_zero`: combined_probability=0.55, ev>0, bankroll=200.0 → asserts `stake_units == 0.0`
- `test_missing_bankroll_fallback`: no bankroll.json → asserts `stake_units == 1.0` (literal fallback, not 0.025)

Tests use `unittest.mock.patch` + `tempfile.TemporaryDirectory` to patch `build_slips.ROOT`, `build_slips.SLIP_DIR`, `build_slips.build_slips` (fixture payload), and `sys.argv`, then call `build_slips.main()` and read the output `slips_<date>.json`. Tests 1 and 2 failed RED against the flat-1.0 placeholder; test 3 correctly passed (fallback behavior already worked).

### Task 2 — GREEN implementation (commit 69a554e)

Modified `scripts/build_slips.py`:

1. Added top-level import: `from stake_sizing import apply_confidence_stakes`
2. Inserted bankroll read + per-category stake application inside `main()`, between `payload = build_slips(...)` and `json_path, md_path = write_outputs(payload, date)`:
   - Reads `ROOT / "data" / "pnl" / "bankroll.json"` defensively
   - Extracts `current_bankroll` with `>0` guard
   - On success: iterates `payload["slips"].items()` and applies `apply_confidence_stakes` per category
   - On any failure (missing file, parse error, value≤0): prints WARNING line, leaves literal `stake_units=1.0`
   - Never calls `apply_confidence_stakes(slips, 1.0)` (that would yield 0.025, not 1.0 — Pitfall 6 avoided)
