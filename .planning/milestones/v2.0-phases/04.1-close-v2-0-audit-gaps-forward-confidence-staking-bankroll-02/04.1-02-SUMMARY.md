## What Shipped

### Task 1: TestSigmaInjection path= override (cc4327e)

`TestSigmaInjection` in `test_weekly_metrics.py` previously redirected the calibration read by patching `self._gp.DATA = tmp_path`. After dedup, the imported `calibration.load_calibration_factor` resolves its path from `calibration.CALIBRATION_PATH` (built from `calibration.ROOT`), not from `generate_projections.DATA` — so the old patch would have silently broken all 6 tests.

Fixed by switching all 6 data-dependent test methods to call `self._gp.load_calibration_factor("SPORT", path=cal_path)` with an explicit path override. This mirrors the existing idiom at lines 449-470 in the same file. `setUp`/`tearDown` simplified (no DATA save/restore needed). All numeric assertions preserved (1.10, 1.0, 1.20, 0.85). The `test_load_calibration_factor_defined_in_module` attribute check still passes because `from calibration import load_calibration_factor` binds the name on the `generate_projections` module.
