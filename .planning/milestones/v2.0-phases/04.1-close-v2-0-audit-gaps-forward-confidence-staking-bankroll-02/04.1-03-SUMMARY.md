### Task 2: D-07 WR-03 Partial Visibility (4 source edits)

**Source A (degraded digest):** In `weekly_metrics_task`, immediately before `send_telegram(digest)`, prepend `"⚠️ WEEKLY METRICS DEGRADED (partial data)\n\n"` when `result.get("status") == "partial"`. Every partial run delivers a visibly degraded Telegram message.

**Source B (JSONL propagation):** In `main()`'s finally block, change `"status": _run_status` to `"status": (_task_result.get("status","ok") if _task_result is not None else _run_status)`. A partial weekly_metrics run now writes `status=partial` to run_log.jsonl (single writer only — Pitfall 5 avoided; `append_run_record` count unchanged).

**Source C (streak counts partial):** Extended `trailing_failure_streak`'s failure set from `{"error","timeout"}` to `{"error","timeout","partial"}`. Two consecutive partial runs produce streak=2, which is >= `REPEATED_FAILURE_THRESHOLD` (2).

**Source D (OBS-03 on success-return path):** On the normal success path in `main()`, after `_task_result = result`, added a check: when `result.get("status") == "partial"`, compute `_obs03_streak = trailing_failure_streak(args.task) + 1` and fire the `🔁 REPEATED FAILURE` alert if `>= REPEATED_FAILURE_THRESHOLD`. Single transient partial (streak=1 < 2) fires no alert and returns 0 (no hard-fail). This wires the alert path that the except branches can't reach for a normally-returning partial.

`test_weekly_metrics.py` additions:
- `TestPartialVisibility`: (1) `test_degraded_digest` — forces aggregation failure, asserts sent digest contains "DEGRADED"; (2) `test_streak_counted` — two prior partial records → streak 2; (3) `test_single_partial_no_hard_fail` — one prior partial → streak 1 < threshold
- `TestReadOnly`: `test_weekly_metrics_does_not_save_master_pnl` — spies on `save_workbook_atomic`, asserts never called during `weekly_metrics_task`
