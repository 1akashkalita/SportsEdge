## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client browser → Flask GET routes | Operator's browser issues GET /, /slips; no request body, no query params consumed |
| workbook content → Jinja-rendered HTML | Player names, selection text, leg strings, reasoning rendered into the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | index.html / slips.html rendering workbook strings | mitigate | Jinja2 autoescaping is on by default in Flask; all double-brace expressions are HTML-escaped; the safe filter is forbidden on any workbook-sourced value (asserted by the `grep -c "| safe"` is 0 acceptance check) |
| T-02-03 | Information disclosure | /, /slips, /history routes | accept | 127.0.0.1 bind enforced in dashboard.py HOST (DASH-03, Phase 1); routes are GET-only, no auth needed for a solo-operator localhost tool (design doc §8) |
| T-02-04 | Tampering | route handlers building workbook paths | mitigate | Routes pass no user input to accessors; get_today_board derives the date from today_str(), not from any query param — no path traversal |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package installs — pure routes + templates over the Phase 1 stack (RESEARCH: no new packages) |
</threat_model>
