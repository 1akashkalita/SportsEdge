## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest 9.0.3 (unittest TestCase style; no fixtures) |
| **Config file** | none — pytest discovers from `scripts/` |
| **Quick run command** | `cd scripts && python3 -m pytest test_stake_sizing.py test_build_slips.py test_weekly_metrics.py -x -q` |
| **Full suite command** | `cd scripts && python3 -m pytest -q` |
| **Estimated runtime** | ~5–15s quick; full suite ~34 min (run only at phase gate) |
| **Known baseline** | "2 failed, 202+ passed" — 2 pre-existing failures in `test_generate_projections.py` are the clean baseline, NOT regressions |

---

## Sampling Rate

- **After every task commit:** Run the quick command above (stake_sizing + build_slips + weekly_metrics)
- **After every plan wave:** Run the touched module's full test files
- **Before `/gsd:verify-work`:** Full suite must be green relative to the "2 failed" baseline
- **Max feedback latency:** ~15 seconds (quick command)
