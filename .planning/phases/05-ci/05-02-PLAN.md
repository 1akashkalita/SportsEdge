```sh
#!/bin/sh
REPO_ROOT="$(git rev-parse --show-toplevel)"
cd "$REPO_ROOT/scripts" || exit 1
python3 run_ci_gate.py

if [ "$status" -ne 0 ]; then
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git push event → pre-push hook | git invokes the committed hook from the repo root on `git push`; the operator is the only user. |
| install_hooks.py → git config | the installer mutates local git config (core.hooksPath) on the operator's own repo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | core.hooksPath already set to something else would be overwritten | mitigate | core.hooksPath is verified UNSET (default .git/hooks/), so setting it is non-destructive; the installer can read it first and report rather than clobber. |
| T-05-05 | Repudiation/Elevation | --no-verify bypasses the gate | accept | Git hooks are inherently bypassable; on the operator's own single-user machine this is an accepted, documented escape hatch (D-discretion). Do NOT attempt to make it unbypassable. |
| T-05-06 | Tampering | committed hook executes on every push | accept | The hook is a thin wrapper the operator authored/reviewed; it runs only the project's own gate. Inherent to any local hook; accepted. |

No HIGH-severity threats. All threats are inherent to a local, single-operator git hook and are accepted with documented reasoning or mitigated by the non-destructive install.
