<objective>
Build the mechanical heart of the CI gate: a fast-subset runner (`scripts/run_ci_gate.py`)
that runs the whole `unittest`/pytest suite EXCEPT a named denylist, behind a fail-loud
environment preflight, plus a dedicated guard test (`scripts/test_ci_environment_guard.py`)
that proves the run-from-`scripts/`-with-`python3` contract and that a project-root `python`
(3.13) invocation fails.

git hook invokes (CI-01) and the artifact that guards the interpreter/CWD footgun (CI-02).
It introduces no `requirements.txt`/lockfile — the guard documents the expected env by
asserting it (D-05).

Output: `scripts/run_ci_gate.py` (gate runner with preflight + denylist), and
`scripts/test_ci_environment_guard.py` (the guard test, auto-included in the gate by D-03).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator's shell → run_ci_gate.py | The operator (sole user) invokes the gate on their own machine; inputs are their own source tree and test files. |
| run_ci_gate.py → pytest child | The gate spawns pytest, which imports and executes the project's own test files. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation/Tampering | pytest child executes arbitrary test code | accept | Running your own test suite executing your own code is an inherent, accepted property of any CI gate on a single-operator machine. No untrusted input crosses the boundary. Not over-engineered. |
| T-05-02 | Information disclosure | guard test spawns a 3.13 `python` child; subprocess could echo env/secrets | mitigate | The negative-case child runs an existing importlib test that does not print secrets; capture output but do not log env. No secret is passed to the child env. |
| T-05-03 | Spoofing | wrong interpreter (3.13) silently used | mitigate | The preflight asserts sys.version_info[:2] == (3, 14) and fails loud before running any test — this is the entire point of CI-02. |

No HIGH-severity threats. The surface is a local test runner on the operator's own machine; threats are inherent to running one's own code and are accepted with documented reasoning.
</threat_model>
