<objective>
Implement OBS-03: when the same task fails twice or more in a row, fire a distinct
`🔁 REPEATED FAILURE` Telegram alert that names the pattern (task, consecutive-failure
count, last error) — in ADDITION to the existing per-occurrence `❌ SPORTS TASK FAILED`
/ `⏱ TASK TIMED OUT` alerts (D-09), never replacing them. The streak is derived from the
shared `run_log.jsonl` tail (D-03) — no separate counter/state file.

Purpose: Turn repeated/patterned failures into qualitatively different signal so an
ongoing outage is unmistakable, while a one-off blip stays a single per-occurrence alert.

Output: A configurable `REPEATED_FAILURE_THRESHOLD` constant (env-driven, default 2), a
trailing-failure-streak helper that reads the run_log.jsonl tail and combines the current
failure, the `🔁` alert wired into both failure branches of `main()`, and a regression
test.
</objective>
