<objective>
Implement OBS-02: a standalone read-only health/heartbeat script
`scripts/health_check.py` that reads `data/pnl/logs/run_log.jsonl` (the OBS-01 record)
and reports, per task, whether it ran within its scheduled window and whether its last
run failed. Dual output per D-06: always print a readable snapshot to stdout; ALSO push
a Telegram alert when any task is overdue or last-ended-in-failure.

Purpose: The operator can run it any time for a system-health snapshot, and it can ALSO
be cron-scheduled as a heartbeat — without acquiring the runner's global fcntl lock
(D-04), so it never blocks on a running task.

Output: `scripts/health_check.py` (standalone, self-contained — no import from the
runner) and `scripts/test_health_check.py`.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering / DoS | JSONL parser (no lock by design, D-04) | mitigate | Parse each line inside try/except json.loads and skip blank/corrupt lines — a partial line written by a concurrent run cannot crash the read-only check; the script never writes the file so it cannot corrupt or partially-write the JSONL |

| T-04-02-03 | Information Disclosure | stdout snapshot | accept | Snapshot prints task names, ages, statuses, and short error strings to the operator's own terminal — LOW, accepted (operator-facing, no new exposure beyond run_log.txt) |
