<objective>
Implement OBS-01: emit one structured JSONL record per runner invocation to
`data/pnl/logs/run_log.jsonl`, with the Core+ field set (D-01/D-02). This is the
single source of truth that OBS-02 (health check) and OBS-03 (streak alert) both
consume, so the record shape must be fixed and correct here.

Purpose: Give the operator after-the-fact structured visibility into every run without
parsing the free-form `run_log.txt`. The structured record lives in a FILE, never an
Excel sheet (locked D-01) — additive observability with zero gate/pick/schema change.

Output: A new `RUN_LOG_JSONL` path constant, an `append_run_record()` serialization
helper, a record-emit call inside `main()`'s `finally` block, and a regression test
proving the record shape and per-outcome status derivation.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| task error/exception → run_log.jsonl `error` field | A caught exception string is serialized to a file on disk; the string could in principle contain secret values if an upstream error embedded a token/URL with credentials |
| run_log.jsonl file on disk | Append-only operator-readable log under data/pnl/logs/; same trust zone as the existing run_log.txt which already records error strings |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | `error` field in run_log.jsonl | mitigate | Serialize only `str(e)` (the same truncatable error string the existing `❌ SPORTS TASK FAILED` alert already sends at line 5728) — do NOT serialize `traceback.format_exc()` or env/secret values into the JSONL record; tracebacks continue to go only to run_log.txt as today |
| T-04-02 | Tampering / Denial of Service | run_log.jsonl write in finally | mitigate | append_run_record is wrapped in try/except Exception: pass so a write failure or disk-full never crashes/fails the task; uses .open("a") append (no overwrite of prior records) |
| T-04-03 | Information Disclosure | run_log.jsonl on disk | accept | File lives under data/pnl/logs/ alongside run_log.txt which already contains error strings; no NEW exposure surface and no secrets are intentionally logged — LOW, accepted with same hygiene as existing logs |
</threat_model>
