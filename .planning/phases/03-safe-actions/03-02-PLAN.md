|----------|-------------|
| caller (route in Plan 03) → dashboard_writes | `date` / `slip_id` form-derived strings cross into the write helper. The helper must treat them as data — used only for cell-value matching, never interpolated into filesystem paths. |
| dashboard_writes → master_pnl.xlsx | The only mutation point. Constrained to additive columns + atomic save on a single fixed path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02 | Tampering | `slip_id` / `date` used in row scan | mitigate | `date`/`slip_id` are compared against cell values only; the write path is a single hardcoded `PNL_DIR / "master_pnl.xlsx"` — form fields are NEVER interpolated into a filesystem path (research Security Domain: path-traversal mitigation). |
| T-03-03 | Tampering | grading-owned `"Notes"` / other sheets | mitigate | add_note writes only the `"Operator Note"` column; mark_placed writes only `Placed`/`Placed At`; only the `Slip History` sheet is opened for write. Proven by ACTION-04c test (`test_write_only_touches_slip_history`). |
| T-03-04 | Denial of Service | concurrent dashboard write vs runner save | mitigate | `workbook_file_lock` cooperative lock + `safe_save_workbook` atomic temp-swap — a mid-write reader/writer sees last-known-good; lock auto-released on exception. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed this phase (03-RESEARCH "Package Legitimacy Audit"); openpyxl/stdlib already present. |

Auth / CSRF: accepted / out-of-scope — single-operator `127.0.0.1`-only tool (REQUIREMENTS.md Out of Scope; design §8).
