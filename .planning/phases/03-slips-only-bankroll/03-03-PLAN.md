<objective>
Re-source the bankroll ledger from slips and decouple it from props (BANKROLL-01 / D-09 / D-13), and add the separate Prop Accuracy summary (BANKROLL-04 / D-10). This adds a new `sync_slip_bankroll(date)` function that owns `bankroll.json` + Daily Log Running Bankroll + Bankroll Chart Data from the master Slip History sheet (excluding `Needs Payout Reconciliation` rows), and severs the prop->bankroll coupling in the existing `sync_master_and_bankroll` so prop grading still records Pick History but no longer touches the bankroll. It also adds a `Prop Accuracy` sheet to `master_pnl.xlsx` via the additive `master_pnl_workbook()` migration, computing prop hit-rate by week and sport without altering Pick History or Results.

Purpose: This is the forward-path machinery — every future daily run computes the bankroll from slips, and props live on purely as a model-accuracy signal. The one-time historical rebuild (03-04) reuses `sync_slip_bankroll`'s aggregation shape and the Prop Accuracy writer.

Output: Edited `scripts/sports_system_runner.py` (new `sync_slip_bankroll`, severed prop coupling, `PROP_ACCURACY_HEADERS` + additive sheet + a `refresh_prop_accuracy` writer) and a new `scripts/test_slip_bankroll.py` with in-memory workbook fixtures.

Wave note: This plan and 03-02 both edit `sports_system_runner.py`; this plan runs in Wave 2 AFTER 03-02 (cap removal) to avoid a same-file collision, and depends on 03-01 for `stake_sizing.confidence_stake` (imported for the day-aggregation it shares with 03-04).
</objective>
