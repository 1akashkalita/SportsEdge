## Directory Purposes

**`scripts/` (source root):**
- Purpose: All Python source code; must be the working directory for every run
- Contains: Orchestrator, subprocess stages, imported helper modules, test files
- Key files: `sports_system_runner.py` (entry point), `workbook_io.py` (I/O safety), `line_timing.py` (Gate 12)
- Note: `scripts/__pycache__/` and `scripts/.pytest_cache/` are generated; `scripts/archive/` is dead code

**`data/nba/` and `data/mlb/`:**
- Purpose: Runtime state — live workbooks and raw prop JSON files
- Contains: Dated `.xlsx` workbooks (one per sport per day), `*_latest.json` pointers (overwritten each run), dated JSON snapshots
- Key files: `data/nba/nba_{YYYY-MM-DD}.xlsx`, `data/nba/prizepicks_nba_latest.json`, `data/nba/dfs_props_unified_nba_latest.json`
- Generated: Yes — created/updated every task run

**`data/pnl/`:**
- Purpose: Aggregate P&L, bankroll state, and operational logs
- Contains: `bankroll.json`, `master_pnl.xlsx`, `run_log.txt`
- Key files: `data/pnl/bankroll.json` (read/written by runner), `data/pnl/logs/run_log.txt` (all log output), `data/pnl/logs/sports_system_runner.lock` (fcntl lock)

**`data/research/`:**
- Purpose: Intermediate computation outputs between subprocess stages
- Contains: Hit-rate JSONs per player, projection JSONs per date, prop monitor snapshots, platform payout config
- Key files: `data/research/projections/{sport}/{sport}_projections_{date}.json` (read by runner after subprocess), `data/research/hit_rates/{sport}/{sport}_{espn_id}_{player}.json`, `data/research/platform_payouts.json` (config for slip_payouts + special_line_value)

**`data/backups/workbooks/`:**
- Purpose: Timestamped backups of every workbook before each save
- Generated: Yes — created automatically by `workbook_io.safe_save_workbook()`
- Committed: No

**`locks/`:**
- Purpose: Cooperative per-workbook lock files (separate from the runner's fcntl lock)
- Generated: Yes — lock files created/deleted at runtime by `workbook_io.workbook_file_lock()`
- Committed: No

**`logs/`:**
- Purpose: Hermes cron scheduler log capture
- Generated: Yes

**`outputs/`:**
- Purpose: Miscellaneous generated files, old audit scripts, dead code
- `outputs/archived_broken_scripts/`: Dead code — do NOT import or build on these files
- `outputs/verification/`: Stress-test outputs and verification scripts

## Naming Conventions

**Files:**
- Source modules: `snake_case.py` (e.g., `fetch_dfs_props.py`, `slip_payouts.py`)
- Test files: `test_{module_name}.py` paired 1:1 with the script under test
- Stage tests: `test_stage{N}_{description}.py`
- Dated data files: `{thing}_{date}.json` or `{thing}_{date}.xlsx` (e.g., `nba_2026-06-13.xlsx`)
- Latest pointers: `{thing}_latest.json` (overwritten each run; points to current data)
- Dated backups: `{workbook}.{HHMMSS}.xlsx` inside `data/backups/workbooks/{YYYY-MM-DD}/`

**Directories:**
- Sport-specific data: `data/{nba,mlb}/` (lowercase)
- Research subdirs: `data/research/{hit_rates,projections,prop_monitor,special_lines,slips,underdog}/`

**Python identifiers:**
- Functions and variables: `snake_case`
- Constants: `UPPER_SNAKE_CASE` (e.g., `PICKS_HEADERS`, `DAILY_EXPOSURE_CAP`)
- Type aliases/classes: `PascalCase` (e.g., `OddsApiIoClient`, `WorkbookAccessError`)

**Dates:**
- Always `YYYY-MM-DD` strings via `today_str()` (no other formats used in file names or workbook rows)
- UTC timestamps via `now_iso()` → ISO 8601 with seconds precision

## Where to Add New Code

**New task function:**
- Implementation: Add a new `def my_task() -> dict[str, Any]:` in `scripts/sports_system_runner.py` near the other task functions (around line 4600)
- Registration: Add to the `mapping` dict in `run_task()` at line 5594
- Locking: Add workbook paths to `task_workbook_paths()` if the task touches any workbook
- Alert: Add a dispatch branch in `dispatch_alerts()` at line 1040
- Test: Create `scripts/test_my_task.py`

**New no-bet gate:**
- Implementation: Add inside `evaluate_no_bet_gates()` at `scripts/sports_system_runner.py:2217`
- Insert before the `return True, None, passed` at the end
- Append to `passed` list on success; return `False, skip_record(...), passed` on failure
- Update `SKIPPED_PICK_HEADERS` if the new gate needs extra columns in Skipped Picks sheet

**New workbook sheet:**
- Add headers list constant near top of `scripts/sports_system_runner.py` (with other `*_HEADERS` constants)
- Add entry to the `expected` dict in `ensure_workbook()` at line 1610
- Schema migration is automatic (additive only)

**New helper module (imported):**
- Create `scripts/{module_name}.py`
- Add import to top of `scripts/sports_system_runner.py` alongside existing imports
- Helper modules must NOT import from `sports_system_runner.py` (circular import)
- Helper modules must be importable when `scripts/` is the working directory

**New subprocess stage:**
- Create `scripts/{stage_name}.py` with a `main()` that outputs JSON to stdout
- Add a `run_{stage_name}()` wrapper in `scripts/sports_system_runner.py` following the pattern of `run_build_hit_rate_db()` at line 1345
- Use `workbook_io.safe_save_workbook()` for any workbook writes inside the subprocess
- Set a timeout appropriate to the operation (300s for fetches, 600s for computation)

**New DFS platform:**
- Add a fetcher in `scripts/fetch_{platform}.py`
- Register it in `scripts/fetch_dfs_props.py` (the aggregator)
- Add a feature flag via `env_bool("ENABLE_{PLATFORM}_PLAYER_PROPS", False)` in `sports_system_runner.py`
- Only PrizePicks and Underdog are first-class (feed gates + picks); new platforms should be comparison-only until validated

**Utilities / shared math:**
- Stateless numeric helpers: add to `scripts/sports_system_runner.py` near `ev_from_probability()` (around line 296)
- I/O safety helpers: add to `scripts/workbook_io.py`
- Platform-agnostic prop helpers: add to `scripts/fetch_dfs_props.py` or `scripts/generate_projections.py`
